# i18n 작업 규칙

`src/messages/` 또는 `src/i18n/` 파일을 수정할 때 반드시 아래 규칙을 따른다.

## 필수 규칙

- `ko.json`을 수정하면 반드시 `en.json`도 동일 키로 수정한다 (반대도 마찬가지)
- 두 파일의 최상위 키 목록은 항상 동일해야 한다
- JSON 들여쓰기는 2칸(space 2)으로 통일

## 키 네이밍 규칙

```
섹션명.항목명          // hero.title
섹션명.항목명.세부항목  // about.values.aiFirst
```

- 섹션명은 컴포넌트 이름과 일치 (camelCase)
- 항목명은 camelCase
- 배열 대신 numbered key 사용 (`feature0`, `feature1`)

## 올바른 ko.json / en.json 예시

```json
// ko.json
{
  "hero": {
    "title": "AI로 게임을 만드는 회사",
    "subtitle": "Codegenic"
  }
}

// en.json — 반드시 동일한 키 구조
{
  "hero": {
    "title": "The AI-Native Game Studio",
    "subtitle": "Codegenic"
  }
}
```

## 새 텍스트 추가 시

`/translate` 스킬을 사용하면 두 파일에 동시 추가된다:

```
/translate hero.cta "게임 보러가기" "See the Game"
```
